# 📦 Git提交指南

## 🎯 快速提交

### 推荐方式：完整提交（包含文档）

```bash
# 1. 添加所有文件（.gitignore会自动排除public等）
git add .

# 2. 检查将要提交的文件
git status

# 3. 提交
git commit -m "Initial commit: Hugo blog with PaperMod theme"

# 4. 添加远程仓库（如果还没有）
git remote add origin https://github.com/yourusername/my-blog.git

# 5. 推送到远程
git push -u origin main
```

## 📋 提交前检查

### 1. 确认.gitignore生效

```bash
# 检查public目录是否被忽略
git status --ignored | grep public
# 应该显示：ignored: public/
```

### 2. 检查文件列表

```bash
# 查看将要提交的文件
git status

# 应该包含：
# ✅ config.yaml
# ✅ .gitignore
# ✅ .gitmodules
# ✅ README.md
# ✅ archetypes/
# ✅ content/
# ✅ static/
# ✅ themes/PaperMod/ (submodule)
# ✅ docs/ (可选)
# ✅ scripts/ (可选)
# ❌ public/ (不应该出现)
```

### 3. 检查文件大小

```bash
# 查看各目录大小
du -sh * .[^.]* 2>/dev/null | sort -h
```

## 🚨 常见问题

### Q: public目录被提交了怎么办？

```bash
# 1. 从Git中删除（但保留本地文件）
git rm -r --cached public/

# 2. 确认.gitignore中有 public/
cat .gitignore | grep public

# 3. 重新提交
git add .gitignore
git commit -m "Remove public/ from git tracking"
```

### Q: 主题文件太大？

如果使用submodule（推荐），主题文件不会直接提交到你的仓库，只会记录submodule的引用。

### Q: 想删除过程文档？

```bash
# 删除文档目录
rm -rf docs/ scripts/

# 提交删除
git add .
git commit -m "Remove documentation and scripts"
```

## 📊 提交大小参考

- 核心文件：~50KB
- 主题（submodule）：~0KB（只记录引用）
- 文档目录：~200KB
- 脚本目录：~5KB

**总计：约250KB**（包含文档）
