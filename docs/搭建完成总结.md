## 🚀 下一步操作

### 1. 修改配置（重要！）

编辑 `config.yaml`，修改以下内容：

```yaml
baseURL: 'https://yourdomain.com'  # ⚠️ 改为你的实际域名
title: '我的博客'                    # ⚠️ 改为你的博客名称
description: '个人技术博客...'      # ⚠️ 改为你的博客描述

# 更新社交链接
socialIcons:
  - name: github
    url: "https://github.com/yourusername"  # ⚠️ 改为你的GitHub
  - name: email
    url: "mailto:your@email.com"            # ⚠️ 改为你的邮箱
```

### 2. 本地预览

```bash
# 启动本地服务器
hugo server -D

# 访问 http://localhost:1313 查看效果
# 按 Ctrl+C 停止服务器
```

### 3. 创建新文章

```bash
# 创建新文章
hugo new posts/my-article.md

# 编辑文章（在 content/posts/my-article.md）
# 记得将 draft: true 改为 draft: false 才能发布
```

### 4. 推送到Git仓库

```bash
# 1. 在GitHub/GitLab创建新仓库

# 2. 添加远程仓库
git remote add origin https://github.com/yourusername/my-blog.git

# 3. 提交并推送
git add .
git commit -m "Initial commit: Hugo blog setup"
git branch -M main
git push -u origin main
```

### 5. 部署到Cloudflare Pages

参考 `cloudflare-pages.md` 文件中的详细步骤：

1. 登录 Cloudflare Dashboard
2. 进入 Pages → Create a project
3. 连接Git仓库
4. 配置构建设置：
   - Framework preset: `Hugo`
   - Build command: `hugo --minify`
   - Build output directory: `public`
5. 保存并部署
6. 配置自定义域名

## 📝 常用命令

```bash
# 创建新文章
hugo new posts/article-name.md

# 本地预览（包含草稿）
hugo server -D

# 本地预览（不包含草稿）
hugo server

# 构建静态文件
hugo --minify

# 查看帮助
hugo help
```

## 🎨 主题自定义

PaperMod主题支持丰富的自定义选项，可以在 `config.yaml` 的 `params` 部分配置：

- 显示阅读时间
- 显示分享按钮
- 显示代码复制按钮
- 显示目录
- 等等...

更多配置选项请参考：[PaperMod文档](https://github.com/adityatelange/hugo-PaperMod)
