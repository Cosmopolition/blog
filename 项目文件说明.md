# 项目文件分类说明

## 📋 文件分类

### ✅ 需要提交到Git的文件（核心文件）

这些是博客运行必需的文件，**必须提交**：

```
blog/
├── config.yaml              # Hugo配置文件（必须）
├── .gitignore              # Git忽略规则（必须）
├── .gitmodules             # Git子模块配置（如果使用submodule）
├── archetypes/             # 文章模板（建议提交）
│   └── default.md
├── content/                 # 内容目录（必须）
│   └── posts/             # 文章目录
├── static/                 # 静态资源（必须）
│   └── [图片、文件等]
└── themes/                 # 主题目录（必须）
    └── PaperMod/          # PaperMod主题
```

### 📚 文档文件（可选提交）

这些是搭建过程的文档，**建议提交**作为参考：

```
docs/
├── README.md                    # 文档目录说明
├── 博客搭建方案.md              # 方案对比文档
├── 快速开始指南.md              # 快速上手指南
├── 搭建完成总结.md              # 搭建总结
├── cloudflare-pages.md          # 部署指南（重要）
├── 部署检查清单.md              # 部署检查清单
└── 解决brew安装问题.md          # 问题解决文档
```

### 🛠️ 工具脚本（可选提交）

这些是搭建过程中的辅助脚本，**可选提交**：

```
scripts/
├── README.md              # 脚本说明
├── install-theme.sh       # 主题安装脚本
└── check-brew-status.sh   # 状态检查脚本
```

### ❌ 不应该提交的文件

这些文件已在 `.gitignore` 中配置，**不会提交**：

```
public/                    # 构建输出（自动生成）
.hugo_build.lock          # Hugo构建锁文件
resources/_gen/           # Hugo生成的资源
.DS_Store                # macOS系统文件
*.log                     # 日志文件
.env                      # 环境变量文件
```

## 📝 提交建议

### 最小提交（仅核心文件）

如果只想提交运行必需的文件：

```bash
git add config.yaml .gitignore .gitmodules
git add archetypes/ content/ static/ themes/
git commit -m "Initial commit: Hugo blog"
```

### 完整提交（包含文档）

如果想保留所有文档作为参考：

```bash
git add .
git commit -m "Initial commit: Hugo blog with docs"
```

### 清理提交（删除过程文档）

如果不需要保留搭建过程文档：

```bash
# 删除docs目录（可选）
rm -rf docs/

# 删除scripts目录（可选）
rm -rf scripts/

# 提交
git add .
git commit -m "Initial commit: Hugo blog (cleaned)"
```

## 🎯 推荐方案

**推荐提交：**
- ✅ 所有核心文件（必须）
- ✅ `docs/cloudflare-pages.md`（部署指南，重要）
- ✅ `README.md`（项目说明）
- ⚠️ 其他文档和脚本（可选，根据个人需求）

这样既保持了项目的整洁，又保留了必要的参考文档。

## 📦 文件大小考虑

- `themes/PaperMod/` - 主题文件较大，但必须提交
- `public/` - 构建输出，不应提交（已在.gitignore中）
- `docs/` - 文档文件较小，可以保留

## 🔍 检查提交内容

提交前可以检查：

```bash
# 查看将要提交的文件
git status

# 查看文件大小
du -sh *

# 检查.gitignore是否生效
git status --ignored
```
